Our Methodology

How we built the brain tumor detection system

Development Process

1
Data Collection & Cleaning

Collected 7,200 MRI images from Kaggle. Removed duplicates, balanced classes, and eliminated overlap between training/testing sets.

2
Data Preprocessing

Resized images to 128×128, applied normalization (0-1 scaling), and implemented data augmentation techniques.

3
Model Architecture

Implemented and compared 6 deep learning models including CNN, VGG16, ResNet50, EfficientNetB0, and MobileNetV2.

4
Training & Evaluation

Trained models with class weighting, early stopping, and learning rate reduction. VGG16 achieved 92.6% accuracy.

5
Deployment

Deployed the best model (VGG16) using Flask web framework with a responsive Bootstrap frontend.

Model Performance Comparison

Why VGG16?

VGG16 was selected as our primary model because:

  • Highest Accuracy: 92.62% on test set
  • Transfer Learning: Pre-trained on ImageNet, reducing training time
  • Deep Architecture: 16-layer network captures complex patterns
  • Reliable Performance: Consistent across all tumor types
